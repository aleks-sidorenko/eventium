# GHC 9.10.3 Upgrade Design

## Goal

Upgrade Eventium from GHC 9.6.7 (Stackage LTS 22) to GHC 9.10.3 (Stackage LTS 24.32). Clean cut — no backward compatibility with GHC 9.6. Library version stays at 0.2.x.

## Key Decisions

- **Target:** GHC 9.10.3, aligned with Stackage LTS 24.32
- **No backward compat:** Drop GHC 9.6 support entirely
- **Absorb x-sum-type-boilerplate:** Inline the ~100 lines of TH into `Eventium.TH.SumType` rather than vendoring as separate package
- **Drop persistent-template:** Merged into `persistent` since 2.12.0.1

## Changes

### 1. Nix Flake & Dev Environment

- Update `flake.nix`: change `pkgs.haskell.packages.ghc967` to the GHC 9.10.3 package set
- Run `nix flake update` for fresh nixpkgs with GHC 9.10.3 support
- Verify `nix develop` loads correctly

### 2. Absorb x-sum-type-boilerplate into eventium-core

The `x-sum-type-boilerplate` package (v0.1.4) has `base < 4.20`, incompatible with GHC 9.10 (base 4.20). Last updated Aug 2024. Rather than fork, absorb the small TH library directly.

**API surface used:**
- `constructSumType` — builds sum types from a list of type names
- `sumTypeConverter` — total conversion between two sum types
- `partialSumTypeConverter` — partial (Maybe) conversion between sum types
- `SumTypeOptions`, `defaultSumTypeOptions`, `SumTypeTagOptions` (ConstructTagName, PrefixTagsWithTypeName, AppendTypeNameToTags), `SumTypeConstructorStrictness`

**Actions:**
- Create `Eventium.TH.SumType` module in eventium-core with the above functions/types
- Re-export from `Eventium.TH`
- Remove `x-sum-type-boilerplate` dependency from `eventium-core/package.yaml` and `examples/bank/package.yaml`
- Update imports in `examples/bank`: `SumTypesX.TH` -> `Eventium.TH`
- Fix any TH API changes needed for GHC 9.10 (template-haskell 2.22)

### 3. Drop persistent-template dependency

`persistent-template` was absorbed into `persistent` at version 2.12.0.1. LTS 24 has persistent 2.17.1.0.

**Actions:**
- Remove `persistent-template` from dependencies in: `eventium-sql-common/package.yaml`, `examples/bank/package.yaml`, `examples/cafe/package.yaml`
- `import Database.Persist.TH` remains unchanged (re-exported from persistent >= 2.14)
- Update persistent bounds: `>= 2.13 && < 2.15` -> `>= 2.14 && < 2.18`
- Update persistent-sqlite and persistent-postgresql bounds similarly

### 4. Dependency Bounds Update

Update all `package.yaml` files to accommodate LTS 24.32 package versions:

| Dependency | Current Bound | New Bound | Reason |
|---|---|---|---|
| template-haskell | < 2.22 | < 2.23 | GHC 9.10 ships TH 2.22 |
| persistent | >= 2.13 && < 2.15 | >= 2.14 && < 2.18 | LTS 24 has 2.17.1.0 |
| persistent-template | >= 2.12 && < 2.13 | (removed) | Merged into persistent |
| persistent-sqlite | >= 2.13 && < 2.14 | Widen for LTS 24 | Check LTS 24 version |
| persistent-postgresql | (test dep) | Widen for LTS 24 | Check LTS 24 version |
| Other deps | Various | Verify against LTS 24 | aeson, mtl, text, etc. |

Run `just hpack` to regenerate all `.cabal` files after changes.

### 5. Build & Fix Compilation

Build bottom-up following dependency order:
1. `eventium-core` (with new `Eventium.TH.SumType`)
2. `eventium-memory`
3. `eventium-sql-common`
4. `eventium-postgresql`, `eventium-sqlite`
5. `eventium-testkit`
6. `examples/counter-cli`, `examples/bank`, `examples/cafe`

Fix any GHC 9.10 warnings/errors (especially new `-Wcompat` warnings).
Run `just format` for ormolu compatibility with GHC 9.10.

### 6. CI & Documentation

- Update `.github/workflows/ci.yml`: GHC 9.10.3, appropriate Cabal version
- Update `CLAUDE.md` to reference GHC 9.10.3
- Full test suite pass (`just test`)

## Risk Assessment

| Risk | Likelihood | Mitigation |
|---|---|---|
| x-sum-type-boilerplate TH code breaks on GHC 9.10 | Medium | Simple TH code, mostly constructors. Fix inline. |
| persistent 2.17 API changes | Low | Well-maintained, TH API stable |
| Nix doesn't have GHC 9.10.3 in nixpkgs | Low | nixos-unstable tracks recent GHCs |
| Other deps incompatible with LTS 24 | Low | allow-newer: true provides safety net during migration |
